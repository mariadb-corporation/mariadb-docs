## New Features

### [MXS-6102](https://jira.mariadb.org/browse/MXS-6102) If a server keeps sending routing replies, label it as Running regardless of monitoring result

A monitored server may remain in running status if the connection failure was
due to connection timeout and if MaxScale has recently received data from the
server in response to a client query. These conditions suggest that the server
is still running, but too busy to process MaxScale monitor connections.

### [MXS-5976](https://jira.mariadb.org/browse/MXS-5976) Add SNI support

Listeners now support [SNI](https://en.wikipedia.org/wiki/Server_Name_Indication).
SNI allows a client to connect to different MaxScale services through one
listener. This feature is configured with an external JSON file. See
[listener documentation](https://app.gitbook.com/s/0pSbu5DcMSW4KwAkUcmX/maxscale-management/deployment/maxscale-configuration-guide.md#sni_mapping_file)
for more information.

### [MXS-5594](https://jira.mariadb.org/browse/MXS-5976) Allow encrypted and unencrypted connections on the same listener

A listeners now accepts encrypted and unencrypted connections if configured with `ssl=allow`.
See [configuration quide](https://app.gitbook.com/s/0pSbu5DcMSW4KwAkUcmX/maxscale-management/deployment/maxscale-configuration-guide.md#tlsssl-encryption)
for more information.
