# MariaDB MaxScale 23.08.13 Release Notes

Release 23.08.13 is a GA release.

**Release Date:** 16 Mar 2026

This document describes the changes in release 23.08.13, when compared to the
previous release in the same series.

If you are upgrading from an older major version of MaxScale, please read the
[upgrading document](https://app.gitbook.com/s/0pSbu5DcMSW4KwAkUcmX/maxscale-management/deployment/upgrading-maxscale)
for this MaxScale version.

For any problems you encounter, please consider submitting a bug
report on [our Jira](https://jira.mariadb.org/projects/MXS).

## Bug fixes

* [MXS-6212](https://jira.mariadb.org/browse/MXS-6212) Kafkacdc doesn't log GTID positions when losing ownership of a cluster
* [MXS-6209](https://jira.mariadb.org/browse/MXS-6209) maxctrl create report may truncate output
* [MXS-6196](https://jira.mariadb.org/browse/MXS-6196) kafkacdc doesn't enable TLS for replication connections
* [MXS-6193](https://jira.mariadb.org/browse/MXS-6193) Crash on malformed OK packet
* [MXS-6103](https://jira.mariadb.org/browse/MXS-6103) MaxScale uses a hard-coded value of LimitNOFILE=65535 in the systemd service file
* [MXS-6093](https://jira.mariadb.org/browse/MXS-6093) maxctrl alter user fails to process purely numeric passwords, resulting in "No password provided" error
* [MXS-6086](https://jira.mariadb.org/browse/MXS-6086) Table name parsing doesn't remove identifier quotes
* [MXS-6043](https://jira.mariadb.org/browse/MXS-6043) MaxScale WebUI: prevent load Adobe CSS for Internet-restricted sites

## Known Issues and Limitations

There are some limitations and known issues within this version of MaxScale.
For more information, please refer to the
[Limitations](https://app.gitbook.com/s/0pSbu5DcMSW4KwAkUcmX/maxscale-management/mariadb-maxscale-limitations-guide)
document.
